# Dino Plat Telemetry (Telegram)

В проект добавлена встроенная клиентская телеметрия для статического хостинга (GitHub Pages):
- уведомление о заходе пользователя;
- данные устройства/браузера (доступные из JavaScript);
- фиксация запуска и завершения игровой сессии;
- отправка статистики: время в игре, счёт, рекорд, количество визитов;
- периодические heartbeat-сообщения во время игры.

## 1) Настройка Telegram-ссылки

1. Откройте файл `telemetry.config.js` в корне проекта.
2. Установите `enabled: true`.
3. Вставьте ссылку в `telegramEndpoint`.

Пример:

```js
window.DinoPlatTelemetryConfig = {
  enabled: true,
  telegramEndpoint: "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/sendMessage?chat_id=<CHAT_ID>",
  heartbeatMs: 60000
};
```

Важно:
- ссылка должна уже содержать `chat_id`;
- скрипт сам добавляет параметр `text` через GET;
- если `enabled: false` или `telegramEndpoint` пустой, отправка отключена.

## 2) Что отправляется

- События: `visit`, `game_start`, `game_pause`, `game_resume`, `game_over`, `heartbeat`, `page_hide`.
- Сводка по пользователю:
  - `visitor_id` (локальный идентификатор в браузере);
  - `visit_count` (сколько раз открывал страницу);
  - `returning_user` (был ли раньше);
  - `total_play_time_ms` (накопленное время игры за все сессии в этом браузере).
- Текущая сессия:
  - `session_play_time_ms`;
  - `score`, `record_score`, `coins`, `diamonds`, `lives`.
- Данные окружения браузера:
  - `userAgent`, `platform`, `language`, `languages`, `timezone`, `screen`, `viewport`, `deviceMemory`, `hardwareConcurrency`, `cookieEnabled`, `online`.
- Telegram WebApp данные (если доступны): `initDataUnsafe.user`.
- URL-параметры игры (`u`, `t`) если переданы.
- Геолокация (если пользователь дал разрешение) — широта/долгота.

## 3) Как это работает в статике

Всё выполняется только в браузере:
- без сервера и без backend;
- отправка в Telegram идёт GET-запросом из JavaScript;
- отправка сделана через `Image` beacon (лёгкий GET и минимальная нагрузка на игру).

## 4) Ограничения

- На GitHub Pages нет доступа к «полностью всем данным»: можно получить только то, что даёт браузер.
- IP/точный регион напрямую JS не видит (без внешнего geo-IP API).
- Геолокация требует разрешения пользователя и может быть недоступна.

## 5) Рекомендации по надёжности

- Не отправляйте слишком длинные тексты (Telegram ограничивает размер сообщения).
- Используйте отдельный бот только под аналитику.
- Учитывайте требования приватности и локального законодательства (уведомление/согласие пользователя при необходимости).
